# Huobao Drama 技术实现（宣讲版）

## 1. 项目定位与价值
- 目标：一站式 AI 短剧生产平台，覆盖剧本生成、角色/道具管理、分镜设计、图/视频生成与合成。
- 设计风格：Go 后端 + Vue3 前端，DDD 分层（API/Handlers → Application Services → Domain Models → Infrastructure），RESTful JSON 接口 + 静态资源统一出口。
- 使用场景：产品宣讲（展示端到端能力、可扩展性）与项目文档归档（技术架构、依赖、部署要点）。

## 2. 技术栈速览
- 后端：Go 1.23、Gin、GORM（默认 SQLite，可切换 MySQL）、Zap 日志、Viper 配置、本地存储（/static）、FFmpeg 封装。
- AI 接入：`pkg/ai` 客户端支持 OpenAI/Gemini/ChatFire/火山豆包/自定义中转等，`ai_service` 统一持久化与模型选择。
- 前端：Vue 3 + TS，Vite 构建，Pinia 状态、Vue Router 路由，Element Plus + Tailwind v4，Axios 封装（/api/v1，10 分钟超时适配生成任务）。
- 交付：多阶段 Dockerfile（Node 构建前端 → Go 构建后端 → Alpine 运行），docker-compose 快速启动，配置示例位于 `.env.example` 与 `configs/config.example.yaml`。

## 3. 架构总览
- 分层目录
  - API 层：`api/handlers` 参数绑定/校验与响应封装；`api/middlewares` 日志、CORS、限流；`api/routes/routes.go` 统一注册 `/api/v1/*`，暴露 `/static`、SPA fallback。
  - 应用层：`application/services` 承载核心业务（剧目/分集管理、AI 配置、脚本与提示词生成、图片/视频生成、资源转存、自动化工作流、任务查询、道具与角色库等）。
  - 领域层：`domain/models` 定义剧目/分镜/资产/任务/时间线/工作流运行等模型与 GORM 映射。
  - 基础设施：`infrastructure/database` 连接与 AutoMigrate；`infrastructure/storage` 本地存储及下载；`external/ffmpeg` 音视频处理封装；`scheduler` 调度。
  - 公共库：`pkg` 下的 AI 客户端、日志、配置、图片/视频工具等。
- 数据与资源
  - API：所有业务接口归口 `/api/v1`（剧目、角色/道具、AI 配置、素材、分镜、任务、音频等）。
  - 静态资源：上传/生成文件通过 `/static` 访问；生产模式托管前端 `/assets`（`web/dist`）。
  - 配置：`configs/config.yaml` 控制端口、CORS、数据库、存储、默认 AI 提供商等；`app.debug` 切换日志模式。

## 4. 核心业务流程（宣讲用文字版流程图）
1) 创建剧目与分集 → 数据入库（`drama`/`episode`）。
2) 触发剧本/分镜生成：前端调用 `/generation/*` 或 `/episodes/*/storyboards`；后端按 AI 配置调模型，生成分镜与提示词入库。
3) 素材生成：调用 `/images`、`/videos` 等批量生成背景、角色图、图生视频；结果落地本地存储并返回可访问 URL/任务状态。
4) 合成与管理：素材可导入资产库、进入时间线或视频合并；FFmpeg 处理音频抽取与合成，成片通过 `/static` 下载。
5) AI Agent 自动化工作流：`auto_workflow_service` 将剧本解析、分镜生成、素材生成、视频合并、任务跟踪串成流水线；支持暂停/恢复、失败重试和进度查询。

## 5. AI Agent 自动化工作流实现
- 编排引擎：`application/services/auto_workflow_service.go` 定义各阶段步骤（剧本生成→分镜→背景/角色图→图生视频→合成导出），状态写入 `workflow_run`/`task` 等模型。
- 异步执行与幂等：任务以数据库记录 + 队列式执行，阶段失败可重试；支持从指定步骤恢复，避免重复生成。
- 调度与控制：路由 `/dramas/:id/workflow/start`、`/resume`、`/status`（通过 `/tasks` 查询）暴露控制面；前端 Auto Workflow 页面可一键启动或断点续跑。
- Agent 能力：根据剧目上下文选择合适的 AI 提供商/模型（文本/图片/视频），并将生成的提示词、分镜、素材 URL 写回数据库供后续步骤消费。
- 资源闭环：生成文件统一落地本地存储并暴露 URL，供时间线/合成环节直接引用；任务状态可视化便于宣讲演示。

## 5. 前端设计要点
- 入口：`src/main.ts` 初始化暗色主题、挂载 Pinia/Router/i18n/Element Plus 全量图标。
- 路由：`src/router/index.ts` 覆盖剧目列表/创建、自动流程、分集剧本编辑、分镜编辑、素材生成、专业编辑器、AI 配置等页面。
- API 封装：`src/api/*.ts` 模块化调用后端 REST；`src/utils/request.ts` 统一超时与错误处理。
- 视图与状态：`src/views` 按业务场景组织；`src/stores` 管理跨页状态（生成任务、素材列表、偏好等）；`src/assets/styles` 自定义主题与样式。

## 6. 运行与部署要点
- 本地开发
  - 后端：`go run main.go`（自动迁移数据库，初始化本地存储）。
  - 前端：`cd web && npm run dev`（端口 3012，代理后端 `/api` 与 `/static` → 5678）。
- 生产与镜像
  - Docker 多阶段：Stage1 构建前端 → Stage2 构建 Go 可执行文件与迁移脚本 → Stage3 Alpine 运行（挂载 `./data` 持久化 DB/文件）。
  - 运行时配置：通过环境或 `configs/config.yaml` 设置存储路径、BaseURL、AI 默认提供商等；默认监听 `:5678`。

## 7. 宣讲亮点（可放 PPT 结尾）
- 全链路自动化：剧本→分镜→素材→合成→成片，一条命令或一键工作流。
- 多模型灵活接入：文本/图像/视频多家厂商可切换，配置可持久化、可排序优先级。
- 快速交付：多阶段镜像一键构建；本地前后端热更新，生产直接托管静态资源。
- 资源可追溯：本地存储 URL 化、任务查询接口、时间线记录，便于复用与追踪。
